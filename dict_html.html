estrutura
<!DOCTYPE html> declaração

Informa ao navegador que o documento usa HTML5. Deve ser a primeira linha de todo arquivo HTML.

<!DOCTYPE html>
<html lang="pt-br">...</html>
<html> raiz

Elemento raiz da página. O atributo lang define o idioma — importante para acessibilidade e SEO.

<html lang="pt-br">
  ...
</html>
<head> metadados

Contém informações sobre a página que não aparecem visualmente: título, charset, links de CSS, meta tags, etc.

<head>
  <meta charset="UTF-8">
  <title>Minha Página</title>
</head>
<meta> metadados

Tag auto-fechante usada dentro do <head>. Define charset, viewport, descrição da página e mais.

<meta charset="UTF-8">
<meta name="viewport"
  content="width=device-width,
           initial-scale=1.0">
<meta name="description"
  content="Descrição da página">
<title> metadados

Define o título exibido na aba do navegador e nos resultados de busca. Essencial para SEO.

<title>Luigi · Portfólio</title>
<body> conteúdo

Todo o conteúdo visível da página vai aqui: texto, imagens, links, formulários, etc.

<body>
  <h1>Olá, mundo!</h1>
</body>
<div> container

Container genérico de bloco. Não tem significado semântico — usado para agrupar elementos e aplicar estilos.

<div class="card">
  <p>Conteúdo aqui</p>
</div>
<span> container

Container genérico inline. Usado para estilizar partes do texto sem quebrar o fluxo da linha.

<p>Texto com
  <span class="destaque">
    palavra
  </span>
em negrito.</p>
texto
<h1> — <h6> heading

Títulos em 6 níveis hierárquicos. Apenas um <h1> por página (SEO). Os demais organizam o conteúdo.

<h1>Título Principal</h1>
<h2>Subtítulo</h2>
<h3>Seção menor</h3>
<p> bloco

Define um parágrafo de texto. O navegador adiciona espaçamento automático acima e abaixo.

<p>Este é um parágrafo.</p>
<p>Este é outro parágrafo.</p>
<strong> inline

Destaca texto com importância semântica. Visualmente exibe em negrito. Diferente de <b> que é só visual.

<p>Não esqueça de
  <strong>salvar</strong>
o arquivo.</p>
<em> inline

Ênfase semântica — aparece em itálico. Usado quando o tom ou significado da palavra muda na frase.

<p>Eu <em>realmente</em>
preciso estudar.</p>
<br> inline

Quebra de linha dentro de um texto. Auto-fechante. Use com moderação — prefira CSS para espaçamento.

<p>Linha 1<br>
Linha 2</p>
<code> inline

Exibe texto em fonte monospace, indicando que é código. Usado inline dentro de parágrafos.

<p>Use
  <code>console.log()</code>
para depurar.</p>
<pre> bloco

Preserva espaços e quebras de linha exatamente como escritos. Geralmente combinado com <code>.

<pre><code>
function ola() {
  return "mundo";
}
</code></pre>
<blockquote> bloco

Representa uma citação longa de outra fonte. O atributo cite pode indicar a URL da origem.

<blockquote cite="https://...">
  <p>"A simplicidade é a
  sofisticação máxima."</p>
</blockquote>
listas
<ul> lista

Lista não-ordenada — exibe itens com marcadores (bullets). Usada quando a ordem não importa.

<ul>
  <li>Python</li>
  <li>JavaScript</li>
  <li>C#</li>
</ul>
<ol> lista

Lista ordenada — numera os itens automaticamente. Útil para passos, rankings ou sequências.

<ol>
  <li>Abrir o terminal</li>
  <li>Clonar o repo</li>
  <li>Rodar o projeto</li>
</ol>
<li> lista

Item de lista — deve ser filho direto de <ul> ou <ol>. Pode conter qualquer conteúdo HTML.

<ul>
  <li>Item simples</li>
  <li><strong>Item
    em negrito</strong></li>
</ul>
<dl> / <dt> / <dd> lista

Lista de definições: <dl> = container, <dt> = termo, <dd> = definição. Ideal para glossários.

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup
    Language</dd>
</dl>
links & mídia
<a> link

Âncora — cria hyperlinks. O atributo href define o destino. target="_blank" abre em nova aba.

<a href="https://github.com"
   target="_blank">
  GitHub
</a>
<img> mídia

Exibe uma imagem. src é o caminho, alt é o texto alternativo (obrigatório para acessibilidade).

<img src="foto.jpg"
     alt="Foto de perfil"
     width="200">
<video> mídia

Embute vídeo na página. Suporta múltiplos <source> para diferentes formatos. controls exibe os controles.

<video controls width="400">
  <source src="video.mp4"
          type="video/mp4">
</video>
<audio> mídia

Embute áudio na página. Funciona da mesma forma que <video> com o atributo controls.

<audio controls>
  <source src="musica.mp3"
          type="audio/mpeg">
</audio>
<iframe> embed

Incorpora outra página ou conteúdo externo (YouTube, Google Maps, etc.) dentro da página atual.

<iframe
  src="https://maps.google.com/..."
  width="400"
  height="300"
  loading="lazy">
</iframe>
<link> metadados

Usado no <head> para vincular recursos externos, principalmente arquivos CSS e fontes.

<link rel="stylesheet"
      href="style.css">
<link rel="icon"
      href="favicon.ico">
formulários
<form> formulário

Container que agrupa campos de formulário. action define para onde os dados vão; method define GET ou POST.

<form action="/submit"
      method="POST">
  ...
</form>
<input> formulário

Campo de entrada. O atributo type define o tipo: text, email, password, checkbox, radio, file, etc.

<input type="text"
       name="nome"
       placeholder="Seu nome"
       required>
<label> formulário

Rótulo de um campo. O atributo for deve corresponder ao id do input — melhora acessibilidade.

<label for="email">Email:</label>
<input id="email"
       type="email">
<textarea> formulário

Campo de texto multilinha. Os atributos rows e cols definem o tamanho inicial visível.

<textarea name="msg"
          rows="4"
          placeholder="Mensagem...">
</textarea>
<select> / <option> formulário

Cria um menu dropdown. <select> é o container e cada <option> é uma escolha disponível.

<select name="pais">
  <option value="br">Brasil</option>
  <option value="fr">França</option>
</select>
<button> formulário

Botão clicável. type="submit" envia o form; type="button" age via JavaScript; type="reset" limpa.

<button type="submit">
  Enviar
</button>
<button type="reset">
  Limpar
</button>
semântica
<header> semântico

Cabeçalho de uma página ou seção. Geralmente contém logo, título principal e navegação.

<header>
  <h1>Meu Site</h1>
  <nav>...</nav>
</header>
<nav> semântico

Marcador de navegação principal. Indica ao navegador e leitores de tela que o conteúdo é um menu de links.

<nav>
  <a href="#home">Home</a>
  <a href="#sobre">Sobre</a>
</nav>
<main> semântico

Conteúdo principal único da página. Deve existir somente um por documento e não deve conter headers/footers repetidos.

<main>
  <h1>Artigo Principal</h1>
  <p>Conteúdo único...</p>
</main>
<section> semântico

Agrupa conteúdo temático relacionado. Diferente de <div>, tem significado semântico e geralmente tem um título.

<section id="projetos">
  <h2>Projetos</h2>
  ...
</section>
<article> semântico

Conteúdo independente e redistribuível, como posts de blog, cards de produto ou comentários.

<article>
  <h2>Post do Blog</h2>
  <p>Conteúdo...</p>
</article>
<aside> semântico

Conteúdo relacionado mas secundário — barras laterais, notas extras, widgets de redes sociais.

<aside>
  <h3>Leia também</h3>
  <ul>...</ul>
</aside>
<footer> semântico

Rodapé de página ou seção. Costuma conter copyright, links secundários e informações de contato.

<footer>
  <p>© 2025 Luigi</p>
</footer>
<figure> / <figcaption> semântico

Envolve conteúdo autônomo (imagem, gráfico, código) com legenda. Ambos se movem juntos no documento.

<figure>
  <img src="grafico.png"
       alt="Vendas 2025">
  <figcaption>
    Vendas por mês
  </figcaption>
</figure>
css · seletores & conceitos
.classe seletor

Seleciona elementos pelo atributo class. Pode ser reutilizado em vários elementos. É o seletor mais comum.

.card {
  background: #fff;
  border-radius: 8px;
}
#id seletor

Seleciona um único elemento pelo atributo id. Deve ser único por página. Alta especificidade.

#header {
  position: sticky;
  top: 0;
}
element seletor

Seleciona todas as tags daquele tipo. Ex: p afeta todos os parágrafos da página.

p {
  line-height: 1.7;
  color: #333;
}
h1, h2 {
  font-weight: 700;
}
:hover / :focus pseudo-classe

Pseudo-classes de estado. :hover = mouse sobre o elemento; :focus = elemento focado (teclado/click).

button:hover {
  background: #0070f3;
}
input:focus {
  outline: 2px solid blue;
}
::before / ::after pseudo-elemento

Inserem conteúdo antes/depois de um elemento via CSS. Requerem a propriedade content.

.titulo::before {
  content: "→ ";
  color: #b08d6a;
}
@media responsivo

Media queries aplicam estilos condicionalmente com base no tamanho da tela, orientação ou tipo de dispositivo.

/* mobile first */
.grid {
  grid-template-columns: 1fr;
}

@media (min-width: 768px) {
  .grid {
    grid-template-columns: 1fr 1fr;
  }
}
display: flex layout

Ativa o Flexbox. Os elementos filhos se tornam flex items, controlados por justify-content, align-items, etc.

.nav {
  display: flex;
  gap: 16px;
  align-items: center;
  justify-content: space-between;
}
display: grid layout

Ativa o CSS Grid — layout bidimensional. grid-template-columns define as colunas; gap controla o espaçamento.

.cards {
  display: grid;
  grid-template-columns:
    repeat(auto-fill, minmax(240px, 1fr));
  gap: 20px;
}
position layout

Controla como o elemento é posicionado: static (padrão), relative, absolute, fixed, sticky.

.overlay {
  position: fixed;
  inset: 0; /* top/right/bottom/left: 0 */
  z-index: 100;
}
var(--custom) variável

Variáveis CSS (custom properties). Definidas com --nome no :root e usadas com var(--nome) em qualquer lugar.

:root {
  --cor-principal: #b08d6a;
  --fonte: 'DM Sans', sans-serif;
}

h1 {
  color: var(--cor-principal);
  font-family: var(--fonte);
}
transition animação

Anima mudanças de propriedades CSS suavemente. Sintaxe: propriedade duração easing delay.

.btn {
  background: #333;
  transition: background 0.25s ease,
              transform 0.2s ease;
}
.btn:hover {
  background: #b08d6a;
  transform: translateY(-2px);
}
@keyframes animação

Define animações CSS personalizadas. Combinado com animation no elemento que deve ser animado.

@keyframes subir {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.card {
  animation: subir 0.5s ease both;
}
